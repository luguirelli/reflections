Para evitar que com autalização automatica o repositorio remoto seja preenchido com commits que eu nao queira.

Describe the differences between forks, clones, and branches.  When would you
use one instead of another?

.Branch são criando em um unico repositorio e sao usado como rotulos para os commits;
    .Clone é usado para criar repositorio identicos/clonar de um repositorio remoto(GitHub) para o Local ou dentro um mesmo Local;
    .Fork é usado para clonar repositorios de GitHub para GitHub.

When would you want to make changes in a separate branch rather than directly in
master?  What benefits does each approach have?

Quando deseja-se fazer modificoes no arquivo porem ainda nao foram analisadas e aprovadas pelo criador do master. Os beneficios sao que com isso pode-se ter varias alterações de um mesmo arquivo e depois é possivel compara-las e escolhe-las conforme a necessidades e gosto, e assim nao perdendo o arquivo original.
